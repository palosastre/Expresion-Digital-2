Tarea:

Reescribir el ejercicio del elemento que rebota junto con el doble for loop, con una sintaxis más limpia. La tarea debe incluir la nueva notación para variables, eliminar comentarios de líneas y reemplazarlos por comentarios para bloques de texto y un cambio en la manera de usar variables en los condicionales. En esta guía será descrito cada uno de los temas.

1) Cambio en la notación de variables

Hasta ahora teníamos una línea de código para una variable y su comentario, desde ahora podemos escribir por grupo de variables separadas por ','.

Lo escribíamos así:


int a; //variable de enteros a
int b; //variable de enteros b
int c; //variable de enteros c
int d; //variable de enteros d
float q; //variable de decimales q
float w; //variable de decimales w
float e; //variable de decimales e
color c; //variable de color c
color d; //variable de color d
color f; //variable de color f

Desde ahora podemos escribir así: Agrupando variables con una línea de comentario ya que sus nombres indican el propósito.


// Posición y movimiento
float posx, posy, movx, movy;

// Colores 
color bl, ng, a, m, y;

2) Cambiar comentarios en líneas por comentarios en bloques de texto

Al pensar la redacción del programa en bloques de texto se hace más fácil la lectura y la búsqueda al momento de modificar una parte del programa, los comentarios como títulos descriptivos de los bloques de texto nos ayudarán ya que contienen todo lo necesario para entender lo que está escrito más abajo. Ya no es necesario que cada comentario de línea describa lo que hace la función, porque hemos aprendido qué significan las funciones.

Muchas veces el comentario supera el ancho de la ventana de Processing, lo hace tedioso de leer.



void setup() {
  size(500, 500); //el tamaño del canvas es 500x500 px

  cn = color(0); //color negro
  cb = color(255); //color blanco
  cr = color(232, 1, 20); //color rojo
  ct = color(6, 255, 192); //color turqueza
  
  posx = width; //variable para la posicion del elipse en el eje x
  posy = height/2; //variable para la posicion del elipse en el eje y, es la mitad de la altura
  movx = 4; //variable para definir el movimiento del elipse; se movera de a 4 pixeles en el eje x en cada frame
  movx = 5; //variable para definir el movimiento del elipse; se movera de a 5 pixeles en el eje y en cada frame
}

Desde ahora podemos escribir así: Agrupando variables con una línea de comentario ya que sus nombres indican el propósito y comentar alguna línea en particular.


void setup() {
  size(500, 500);

  // Colores
  cn = color(0); //negro
  cb = color(255); //blanco
  cr = color(232, 1, 20); //rojo
  ct = color(6, 255, 192); //turqueza

  // Inicialización de posición y movimiento
  posx = width; 
  posy = height/2;
  movx = 4;
  movx = 5;
}

2.1) Cambiar comentarios en líneas por comentarios en bloques de texto en

void draw

El void draw es en donde más código estaremos escribiendo en esta parte del semestre, llevar un orden a través de comentar con forma de título es importante para una buena redacción y entendimiento de lo escrito.

Al igual que en el void setup () teníamos agrupados por bloques de texto pero con comentarios por cada línea de código:



void draw() {
  background(cn); //color celeste de fondo del canvas 
  fill(m); //relleno elipse color blanco
  ellipse(posx, posy, 60, 60); //ubicación inicial de la elipse y su tamaño es de 60px x 60px sin stroke

  posx = posx + movx; //la posición de la elipse en el eje x es igual a la posición en x más 8 px por cada frame
  posy = posy + movy; //la posición de la elipse en el eje y es igual a la posición en y más 5 px por cada frame

  if (posx < 0) { //si la posición de x es menor a cero 
    dx = - dx; //la dirección de x es igual a la dirección x por menos uno (Asi la elipse se mueve en sentido  contrario)
    fill(c); //relleno elipse color naranjo (Cambia asi cuando toca el borde del lado izquierdo)
    stroke(m); //borde del elipse color naranjo (Cambia asi cuando toca el borde del lado izquierdo)
    strokeWeight(4); //borde de la elipse es de grosor de 40 px (Cambia asi cuando toca el borde del lado izquierdo)
  }
}

Podemos ir separando los bloques de texto entre dibujo, movimiento, condicionales, etc. y comentarios de línea en funciones que quisieramos tener más atención:


void draw() {
  background(cn);

  // ellipse de color magenta y borde amarillo de 2
  stroke(a);
  strokeWeight(2);
  fill(m);
  ellipse(posx, posy, 60, 60);

  // velocidad
  posx = posx + movx;
  posy = posy + movy;

  // si posx es menor a 0, cambia dirección, fill cyan, stroke magenta de 4
  if (posx < 0) {
  posx = -posx;
  fill(c);
  stroke(m);
  strokeWeight(4);
  }
}

3) Cambiar la forma en que se usan las variables condicionales

if (test) { statements }

En todos los ejercicios que han hecho, ocurre una mezcla entre variables y funciones usadas. Si la condición es verdadera, los statements deben reflejar los cambios que se quieren hacer sin caer en una redundancia. En el ejemplo del elemnto que rebota en los bordes del canvas ocurre la mezcla: 


// si posx es mayor al ancho, cambia dirección, fill cyan, stroke magenta de 4
if (posx > width) {
  movx = -movx;
  fill(a);
  stroke(m);
  strokeWeight(4);
  ellipse(posx, posy, 60, 60);
}

El problema ocurre en los statements, en donde no se cambian los argumentos de las funciones, si no que se sobre escriben nuevan funciones con nuevos argumentos, muchos tienen este problema, en donde realizan cambios y aún así el elemento no cambia de color ó tamaño, entonces lo que hacían era solamente declarar el elemento sin color y ahí sí las condicionales funcionaban.


background(0);
fill(m);
stroke(ng);
strokeWeight(1);
ellipse(posx, posy, 20, 20);

// si posx es mayor al ancho, cambia dirección, fill cyan, stroke magenta de 4
if (posx > width) {
  movx = -movx;
  fill(a);
  stroke(m);
  strokeWeight(4);
  ellipse(posx, posy, 60, 60);
}

Pero la inicialización del objeto era con los colores por defecto de Processing.


background(0);
ellipse(posx, posy, 20, 20);

// si posx es mayor al ancho, cambia dirección, fill cyan, stroke magenta de 4
if (posx > width) {
  movx = -movx;
  fill(a);
  stroke(m);
  strokeWeight(4);
  ellipse(posx, posy, 60, 60);
}

Para que sí ocurran los cambios que queremos en el método if debemos hacer un proceso de dos pasos:

Crear una nueva variable específica para cada función.

Usar esta variable de la misma forma en que hemos usado la variable de movimiento para modificar la posición.


//declaramos la variables ya existentes
color m, a, y;


//declaramos la nueva variable con un nombre que describa su proposito
color fillEllipse; 

void setup() {

  //inicializamos los colores
  c = color(0, 255, 255); // cyan
  m = color(255, 0, 255); // magenta
  y = color(255, 255, 0); // amarillo

  //asignamos un color a la variable que guardará el color en la función
  fillEllipse = c;
}

void draw() {
  //asignamos la nueva variable a la funcion que queremos
  fill(fillEllipse);
  ellipse(posx, posy, sizeE, sizeE);

  //finalmente en el condicional nos preocupamos de asignarle 
  //una nueva variable (de la lista de variables) a la variable especial
  if (posx > width) {
    fillEllipse = a;
  }
}

De ésta forma tendremos escrito funciones de dibujo en bloques de texto, el pricipal y modificaremos los valores de sus argumentos en otros bloques de texto como puede ser en el caso de usar el método if.
